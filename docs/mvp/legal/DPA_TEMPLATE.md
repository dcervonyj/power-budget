## 3. Processor Obligations (Art. 28 GDPR)

The Processor shall:

3.1 Process personal data only on documented instructions from the Controller.

3.2 Ensure that persons authorised to process the personal data have committed to confidentiality.

3.3 Implement appropriate technical and organisational measures (Art. 32 GDPR), including:
- Encryption at rest and in transit
- Access controls and audit logging
- Regular security assessments

3.4 Not engage another processor (sub-processor) without prior written authorisation from the Controller.

3.5 Assist the Controller in responding to data subject rights requests.

3.6 Delete or return all personal data to the Controller after the end of the services.

3.7 Make available all information necessary to demonstrate compliance with Art. 28 GDPR.

## 4. Controller Obligations

The Controller shall:

4.1 Provide lawful instructions to the Processor.

4.2 Ensure a valid legal basis exists for processing before instructing the Processor.

4.3 Notify the Processor of any data subject rights requests that require the Processor's involvement.
